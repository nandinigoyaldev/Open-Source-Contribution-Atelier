# 📜 Reading Project Guidelines

### Before you start writing code or opening issues, you must understand the project's rules. These are always laid out in two critical files:

---

### 📖 README.md (Read Me File)

The entry point of the project. It outlines:

- What the project does.
- Key features of the project.
- How to install/clone and run it locally.
- Quick usage examples.

---

### 🤝 CONTRIBUTING.md (Guide to Contributing)

The contributor's playbook. It explains:

- The codebase architecture.
- Preferred branch naming styles (e.g. `fix/...` vs `bug/...`).
- How to run tests and linters locally.
- Pull Request / Issue submission templates,
- What portion of the project not to touch/modify,

> [!WARNING]
> Submitting code that violates the style guide or fails existing tests because you didn't read `CONTRIBUTING.md` wastes maintainers' time. Always inspect guidelines first!
